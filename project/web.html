Web Basic Concepts

HTML / CSS / JavaScript

우선 웹 개발을 하기 위해서는 다음의 3가지 언어가 무엇인지 알아야 했다.

  • HTML : define the content of web pages
  • CSS : specify the layout of web pages
  • JavaScript : program the behavior of web pages
간략히 집을 짓는 것에 비유하여 정리하자면,
  • HTML은 웹 페이지의 구조를 설계한다.
  • CSS는 이렇게 구조화된 것에 인테리어를 하듯이 style을 추가해 디자인을 입힌다.
  • JavaScript는 특정 요구에 맞춰 HTML과 CSS로 지어진 집에 변화를 준다.
결과적으로 웹 페이지란, HTML/CSS/JavaScript로 쓰여진 페이지에 대한 설계도라 볼 수 있다.

그렇다면 이 설계도를 어디에 쓸까?

Web Browser

설계도가 있다고 우리는 집을 볼 수 없다.
마찬가지로 HTML 문서가 있다고 해서 우리는 이를 평소에 보는 웹 페이지처럼 볼 수 없다. 이를 해결해주는 것이 우리가 평소에 사용하는 Chrome이나 IE와 같은 Web Browser이다.

web browser는 html 문서를 읽어, 약속된 형태로 브라우저 화면에 랜더링해주는 기능을 지니고 있다.
브라우저에서 F12를 누르면 브라우저가 랜더링하고 있는 .html을 살펴볼 수 있다.

Server/Client

그렇다면 평소에 우리의 컴퓨터에서 웹 브라우져를 통해 다양한 웹 페이지를 볼 수 있다.
그럼, 이런 웹 페이지들의 html 문서는 어디서 얻는 것일까?

우리가 어떤 웹 사이트를 보기 위해서는 사이트 주소(url)를 입력한다.
우를 통해 우리는 웹 서버에 url에 해당하는 html 문서를 얻고, 브라우저가 이를 랜더링해주어 웹 페이지를 볼 수 있다.

즉, 우리는 web client로 pc나 mobile에서 web server에 접속해 html 문서를 요청하고, 이를 받아와 웹 브라우저로 랜더링하여 웹 페이지를 보는 것!

Responsive, Static/Dynamic

web에 대해 찾아보면 반응형(responsive) 웹, 정적(static) 웹/동적(dynamic) 웹으로 web을 지칭하는 경우가 있다.
대략적으로 살펴본면 다음과 같을 것 같다.

  • 반응형 웹 : 웹 브라우저의 상태(크기, 클릭 등)에 따라 웹 페이지가 변화하는 형태로 쓰여진 웹
  • 정적 웹 : 웹 서버로부터 미리 정해진 형태의 웹 문서만을 받아오는 형태의 웹
  • 동적 웹 : 웹 서버로부터 동적으로 변경된 문서를 받아 유저의 요청에 동적으로 페이지가 변화하는 웹
그렇다면 내가 만드려는 웹은 어떤 웹이 되어야 할까?

Web Architecture

web concept

구상하는 웹 페이지는 개인용 블로그로 딱히 동적인 기능을 요하지 않는다.
따라서 정적인 페이지로 만들기로 결정하였고, +@ 로 가능하면 반응형으로 다양한 환경에서 보기좋은 디자인이 되었으면 했다

web hosting service

이를 위해서는 static web을 서비스할 수 있는 웹 서버가 필요했다.
이 경우에는 이미 많은 사람들이 사용하고 있고, 개인적으로 사용이 익숙한 github에서 제공하는 github page라는 웹 호스팅 서비스를 사용하기로 했다

static site generator

정적인 웹 사이트를 간단하게 만들고 관리하는데 도움이 되는 서비스로 static site generator가 있었다.
이는 웹 페이지의 많은 부분을 템플릿으로 만들어 놓고, 글을 쓸 경우 자동으로 웹 페이지를 조립해 웹 사이트를 만들어주는 기능을 가지고 있다.
관련해서는 다음을 참고하였다.

개인 블로그를 만드는 것이여서, 성능적인 부분은 크게 구애받지 않았다.
다만 그 중에서도 관심이 있던 golang으로 만들어진 Hugo가 마음에 들어 살펴보았다.
정해진 directory structure 안에 미리 theme를 만들어두고, golang으로 scripting하여 페이지를 조립해주는 기능을 하는 것 처럼 보였다.
하지만 아직 (관심은 있지만..) 공부해보지 않음 go로 작성해야 하고, 제공되는 테마가 부족해 마음에 드는 theme를 찾지 못했다.
따라서 static site generator를 이용한 posting 자동화는 미뤄두었다.

front-end의 기본이 되는 HTML, CSS, JavaScript로 직접 구현해보는데 초점을 두었다
관점에서 웹페이지를 만들어 보기로 하였다.

site implementation

site를 구현하기 위해 cover에서 소개한 생활코딩과 w3schools를 따라 기초 웹 코딩을 실습해 보았다.
또한 모든 ui를 직접 디자인하기에는 무리가 있어 Bootstrap4라는 web UI framework를 사용하였다.
그리고 실제 구현에는 다양한 web page들의 구현 코드를 살펴보고 마음에 드는 부분을 차용하는 형식으로 작업을 하였다.
주로 참고한 theme는 에는 기존에 구현되어 있는 다양한 theme를 살펴ㅂ
